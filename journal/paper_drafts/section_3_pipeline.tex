\section{Method: PSM as MLLM Prefilter}
\label{sec:pipeline}

\psm{} (\S\ref{sec:method}) returns ranked
$(t, \mathrm{cell}, \mathrm{lat}, \mathrm{lng}, \mathbf{e})$
tuples---not natural-language answers.
For look-back QA we therefore use \psm{} as a \emph{prefilter}
that narrows the search space, and a frontier MLLM as the
\emph{reranker} that picks the localized answer.

\subsection{Pipeline}
\label{sec:pipeline-stages}

Three stages address distinct failure modes.
Figure~\ref{fig:f1-architecture} sketches the end-to-end flow.

\input{figures/f1_architecture.tex}

\paragraph{Stage~1: PSM top-$K$ retrieval.}
Embed the question $q$ with the ingest encoder and call
\texttt{SpatialMemory\_query\_similar} with the configured
\caponek{} and $K{=}5$.
The call returns five candidates sorted by cosine similarity; the
latency numbers reported for this call come from host-CPU systems
benchmarks in \S\ref{sec:results-memory-latency}, not from an
on-device wearable trace.
Encoder choice is a pipeline knob, not an architectural commitment:
the same path serves three families---CLIP-L (430\,M, 768\,d),
CLIP-bigG (2.5\,B, 1280\,d), and
SigLIP-2-large~\cite{tschannen2025siglip2}
(660\,M, 1024\,d)---via a single factory;
the C engine sees only a $D$-dimensional float vector and is
encoder-agnostic.
Cross-encoder generalization in \S\ref{sec:results-multi-corpus}
confirms the spatial-axis behavior is not a CLIP-L artifact.

\paragraph{Stage~2: Frame fetch.}
Each candidate $t_i$ identifies a frame decoded on-demand from the
source MP4 or VRS stream; in deployment the frame already resides
in the device cache, so the cost reduces to a memcpy rather than a
video seek.

\paragraph{Stage~3: MLLM rerank.}
The five base64-encoded frames plus the question are sent in one
API call to Gemini 3.1 Pro~\cite{google2026gemini}.
The frozen prompt asks the MLLM to select the single frame index
from the $K$ candidates that best answers the question.
The parsed integer reorders \psm{}'s top-$K$ so the chosen frame
occupies position~1 and the remaining four fill positions 2--$K$.

\subsection{Architectural Decomposition}
\label{sec:pipeline-rationale}

\psm{} contributes \emph{recall}: at the bounded $R{=}128$ operating
point its top-5 reaches 21/187 narrations vs.\ brute-force CLIP's 25/187,
while the $R{=}1024$ oracle row recovers the same 25/187
(\S\ref{sec:results-main}).
The MLLM contributes \emph{localization precision}: adding Gemini
at \caponek{}=$K$ lifts exemplar mIoU@5 by 37\,\% relative.
At \caponek{}=1 + Gemini, mIoU reaches 0.069---statistically
indistinguishable from brute-force 0.074
(paired bootstrap $\Delta{=}{-0.005}$, 95\,\% CI
$[-0.032,\,+0.021]$, $n{=}187$).
